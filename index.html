Recommendation: Use Separate Repositories

For Next.js + Python deployed on GCP, separate repos is the better long-term choice

✅ Why Separate Repos Wins

  • Cleaner, isolated deployments per service
  • Simpler, faster CI/CD pipelines
  • Faster builds — only changed service rebuilds
  • Better service separation and ownership
  • Easier access control per repository
  • Scales naturally as your team grows

🗂 When Monorepo Makes Sense

  • Very early stage with 1–2 developers
  • Heavy code sharing between frontend/backend
  • Atomic cross-service commits needed
  • Team comfortable with complex pipelines
  • Shared type definitions or schema files
